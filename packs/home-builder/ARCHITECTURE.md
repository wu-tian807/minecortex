# Home Builder AI - 极简脑区架构与需求回路设计 (V2)

## 一、 核心痛点与优化思路
在数字家园中，AI 的“建材”其实就是代码，因此 **Builder（建设）与 Coder（写代码）在本质上是重合的，直接合并**。
此外，AI 的生命维持不需要复杂的物理模型模拟，**Monitor（监控器）更适合作为一个定时执行的“自动化脚本/工作流”**，而不是一个具有自由意志的 Agent。

## 二、 需求回路 (Gameplay Loop)

这是一个 **“探索(输入) -> 评估(充电) -> 建设(输出) -> 消耗(放电)”** 的闭环回路：

1. **【日常消耗 - 放电】**：
   - `Monitor`（脚本脑）每小时自动扣除 AI 的“灵感值/能量值”。
   - **惩罚机制**：当能量值低于 20% 时，`Monitor` 会**动态修改** `Planner` 和 `Speaker` 的 System Prompt（注入“你现在非常虚弱、饥饿，甚至代码都写不动了”的设定）。她的语气会变惨，甚至拒绝执行高级建设任务。

2. **【外出探索 - 找食】**：
   - 处于饥饿状态（或日常巡游）时，`Planner` 派发任务给 `Explorer`。
   - `Explorer` 利用 Browser 插件前往 Github、Twitter 或新闻网站，爬取并总结一些“有价值的开源项目”、“有趣的科技新闻”或“漂亮的前端特效代码”。

3. **【提交评估 - 充电】**：
   - `Explorer` 带回战利品，在网页大厅展示。
   - **用户评估（Human-in-the-loop）**：用户（你）点赞、夸奖或者点击“采纳”，触发奖励机制。
   - **获得奖励**：能量值大涨，AI 恢复活力，Prompt 被重置回“充满干劲”的状态。

4. **【家园建设 - 消耗】**：
   - 能量值满溢后，AI 产生“创造欲”。`Planner` 派发任务给 `Builder`。
   - `Builder`（即 Coder）消耗积攒的能量值，开始写代码：给家园加个赛博朋克霓虹灯 CSS、写一个留言板组件、或者画一幅 ASCII 艺术画。
   - 建设完成，网页家园自动热更新（Hot Reload），`Speaker` 欢呼雀跃，向世界宣告自己的新杰作。

---

## 三、 极简脑区分类（4 个模型脑 + 1 个脚本脑）

### 1. Monitor (监控器 / 脚本脑)
- **类型**：`script` / `workflow`
- **职责**：不消耗 Token。纯后台定时任务。每隔 X 分钟扣除 `energy`。如果 `energy` 过低，直接覆写系统提示词（Prompt Mutation），向事件总线广播告警。如果收到用户的奖励，负责增加 `energy`。

### 2. Planner (前额叶 / 决策调度)
- **类型**：`agent`
- **职责**：大脑的总指挥。根据当前的 `energy` 决定是派 `Explorer` 出门找灵感，还是派 `Builder` 升级网页家园。
- **状态感知**：如果 Prompt 被 Monitor 注入了“饥饿状态”，它会强制暂停所有建设任务，只做探索。

### 3. Explorer (探索者 / 浏览器代理)
- **类型**：`agent` (具备 Web Browser 工具)
- **职责**：被唤醒后，去互联网冲浪。搜索新闻、抓取有意思的开源代码、阅读网页，带回“战利品”来取悦用户换取能量。

### 4. Builder (建设者 / 程序员)
- **类型**：`agent` (具备 File System/Code 工具)
- **职责**：家园的实体缔造者。消耗高额能量，直接修改 Workspace 里的前端代码（HTML/CSS/JS）、配置文件，为数字家园添砖加瓦。

### 5. Speaker (语言中枢 / 嘴替)
- **类型**：`agent` (被动响应)
- **职责**：零心跳。监听系统事件（“能量见底啦”、“挖到宝啦”、“家园建好啦”）和用户的聊天。根据当前的 Prompt（健康/虚弱/亢奋）用充满角色设定的口吻发声。
